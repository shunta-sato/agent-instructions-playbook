# Narrative/problem-framing note contracts

Use these contracts for paraphrased narrative/problem-framing learning notes. Do not copy long passages from the source. Prefer paraphrased explanation, compact references, and original learning structure. Use short quotations only when necessary and only within applicable copyright limits.

## Part/gate note

Purpose: group cases by the framing problem they train.

Required content:

- part theme or gate
- recurring apparent problem
- hidden frame pattern
- reframing capability the reader should practice
- links to cases, heuristics, diagnostic questions, worksheets, and review notes

## Case/story note

Purpose: preserve the learning mechanics of a case without transcribing it.

Required content:

- apparent problem
- actor/problem owner
- hidden frame
- ownership or origin of the frame where relevant
- reframing move
- consequence
- reader diagnostic question
- links to heuristics and worksheets

Failure modes:

- quote-list reduction
- plot-summary-only case
- missing apparent problem
- missing hidden frame
- missing reframing move
- missing reader application question

## Heuristic/principle note

Purpose: convert a case lesson into a usable diagnostic question.

Required content:

- heuristic written as a question or testable prompt
- case situations where it applies
- hidden frame it exposes
- misuse or overgeneralization risk
- next reader action
- links to cases, diagnostic questions, and worksheets

## Diagnostic question note

Purpose: help a reader detect whether the same framing trap is present in their situation.

Required content:

- question text
- apparent problem it tests
- hidden frame signal
- evidence to gather
- possible reframing move
- links to cases, heuristics, and worksheets

## Worksheet/application note

Purpose: turn a narrative lesson into a reusable application template.

Required content:

- situation prompt
- apparent problem field
- actors/problem owners field
- hidden frame hypotheses
- reframing move candidates
- consequence or decision field
- next action
- links to source cases and heuristics

## Review/index note

Purpose: support navigation, recall, and transfer.

Required content:

- source classification
- case and heuristic map
- diagnostic question index
- worksheet index
- review prompts that require applying reframing moves
- links to part/gate notes and mechanical gate artifacts when relevant
